# Slide Type Templates

Detailed templates for each common slide type in presentation decks.

## 1. Title Slide

**Purpose**: First impression, sets tone and context

### Required Elements
- Company/project name (largest text)
- Tagline or value proposition (2nd largest)
- Date or version (small, footer)
- Logo (if applicable)

### Visual Approaches
- **Hero Image**: Full-bleed background with overlay text
- **Minimal**: Solid color background with centered typography
- **Split**: Image on one side, text on other
- **Abstract**: Geometric or gradient background

### Prompt Template
```
Title slide for [PROJECT NAME]. [TAGLINE/VALUE PROP].
Background: [COLOR/STYLE].
Logo: [POSITION if applicable].
Modern, professional, [MOOD] aesthetic.
16:9 aspect ratio, high contrast typography.
```

### Example
```
Title slide for "Project Apollo" - "PHP-Backed Digital Cash for Modern Remittance".
Background: Dark navy (#0a1628) with subtle tech pattern.
Logo: Bottom right corner.
Modern, professional, tech-forward aesthetic.
16:9 aspect ratio, magenta accent color (#e91e63).
```

## 2. Problem Statement

**Purpose**: Create urgency, establish need for solution

### Required Elements
- Clear problem headline
- 2-4 pain points
- Optional: Statistics that quantify the problem
- Visual representation of pain

### Visual Approaches
- **Icon Grid**: Each pain point with representative icon
- **Statistics Focus**: Large numbers with context
- **Before State**: Visual showing current broken state
- **Cost of Inaction**: Financial impact visualization

### Content Principles
- Lead with the audience's pain, not your solution
- Use concrete numbers over vague claims
- Make it relatable to decision-makers

### Prompt Template
```
Problem statement slide showing [NUMBER] key pain points.
Headline: "[PROBLEM HEADLINE]"
Pain points: [LIST WITH ICONS/VISUALS]
Style: [INFOGRAPHIC/MINIMAL/DRAMATIC]
Background: [COLOR], icons in [ACCENT COLOR].
16:9 aspect ratio.
```

## 3. Solution Overview

**Purpose**: Present your answer to the established problem

### Required Elements
- Solution name/concept
- High-level description (1-2 sentences)
- Key differentiators (2-4 points)
- Visual representation

### Visual Approaches
- **Product Screenshot**: If software, show the interface
- **Diagram**: System architecture or flow
- **Before/After**: Transformation visualization
- **Feature Grid**: Icons with key capabilities

### Content Principles
- Connect directly to problems stated earlier
- Keep it high-level (details come later)
- Focus on outcomes, not features

### Prompt Template
```
Solution slide introducing [SOLUTION NAME].
Headline: "[SOLUTION HEADLINE]"
Description: "[1-2 SENTENCE SUMMARY]"
Key points: [3-4 DIFFERENTIATORS]
Visual: [SCREENSHOT/DIAGRAM/ILLUSTRATION]
Background: [COLOR], accent: [COLOR].
```

## 4. How It Works

**Purpose**: Explain the mechanism or process

### Required Elements
- Step-by-step flow (3-6 steps ideal)
- Clear labels for each step
- Visual connections between steps
- Optional: Timing or phase labels

### Visual Approaches
- **Horizontal Flow**: Left-to-right process
- **Vertical Timeline**: Top-to-bottom steps
- **Circular Process**: Continuous/cyclical workflows
- **Hub and Spoke**: Central system with connections

### Content Principles
- Maximum 6 steps (cognitive limit)
- Use action verbs for labels
- Show, don't tell

### Prompt Template
```
Process flow diagram showing [NUMBER] steps.
Steps: [STEP 1] → [STEP 2] → [STEP 3]...
Style: [HORIZONTAL/VERTICAL/CIRCULAR] flow.
Icons: [STYLE] for each step.
Background: [COLOR], arrows in [ACCENT].
16:9 aspect ratio, clear labels.
```

## 5. Benefits / Value Proposition

**Purpose**: Articulate the value to the audience

### Required Elements
- 3-6 key benefits
- Each with icon and label
- Optional: Supporting metrics

### Visual Approaches
- **Card Grid**: Benefits in individual cards
- **Icon Row**: Horizontal icons with labels
- **Numbered List**: Prioritized benefits
- **Comparison**: With vs. without

### Content Principles
- Benefits > Features (what they get, not what it does)
- Quantify where possible
- Prioritize by audience importance

### Prompt Template
```
Benefits slide with [NUMBER] key value propositions.
Benefits: [BENEFIT 1], [BENEFIT 2], [BENEFIT 3]...
Each with [ICON TYPE] icon.
Layout: [GRID/ROW/LIST].
Background: [COLOR], icons: [ACCENT].
```

## 6. Social Proof / Testimonials

**Purpose**: Build credibility through third-party validation

### Required Elements
- Quotes, logos, or case study results
- Attribution (name, title, company)
- Optional: Photos or company logos

### Visual Approaches
- **Logo Grid**: Partner/client logos
- **Quote Card**: Single powerful testimonial
- **Results Grid**: Multiple case study metrics
- **Timeline**: History of partnerships

### Content Principles
- Relevance to audience (similar industry/size)
- Specific results over general praise
- Visual recognition (known logos)

### Prompt Template
```
Social proof slide showing [TYPE: logos/testimonials/results].
Content: [LOGOS LIST] or "[QUOTE]" - [ATTRIBUTION]
Layout: [GRID/CENTERED/TIMELINE].
Background: [COLOR], subtle professional style.
```

---

## 7. Team / About

**Purpose**: Humanize the company, build trust

### Required Elements
- Key team members (photos or icons)
- Names and titles
- Optional: Credentials or background

### Visual Approaches
- **Photo Grid**: Headshots with names
- **Icon Representation**: Stylized avatars
- **Timeline**: Company history
- **Credential Focus**: Logos of past employers/education

### Content Principles
- Include only decision-relevant people
- Highlight relevant experience
- Keep it brief (this isn't the focus)

---

## 8. Metrics / Traction

**Purpose**: Demonstrate momentum and results

### Required Elements
- Key performance indicators
- Growth trends
- Comparative context

### Visual Approaches
- **Big Numbers**: Large statistics with labels
- **Charts**: Line/bar graphs showing growth
- **Gauges**: Progress toward goals
- **Comparison**: Industry benchmarks

### Content Principles
- Choose metrics that matter to audience
- Provide context (%, vs. industry)
- Show trajectory, not just snapshot

### Prompt Template
```
Metrics slide with [NUMBER] key statistics.
Stats: [METRIC 1: VALUE], [METRIC 2: VALUE]...
Style: [BIG NUMBERS/CHARTS/GAUGES].
Background: [COLOR], numbers in [ACCENT].
Professional, data-forward aesthetic.
```

---

## 9. Competitive Advantage

**Purpose**: Differentiate from alternatives

### Required Elements
- Comparison criteria
- Your position vs. others
- Clear winner indication

### Visual Approaches
- **Comparison Matrix**: Features vs. competitors
- **Positioning Chart**: 2x2 grid placement
- **Feature Comparison**: Checkmarks/X marks
- **Unique Differentiator**: Single point focus

### Content Principles
- Be fair but favorable
- Focus on criteria that matter to audience
- Don't name competitors if sensitive

---

## 10. Roadmap / Timeline

**Purpose**: Show future plans and phases

### Required Elements
- Phases or milestones
- Timing (quarters, months, weeks)
- Current position indicator

### Visual Approaches
- **Horizontal Timeline**: Left-to-right progression
- **Vertical Roadmap**: Top-to-bottom phases
- **Gantt-style**: Overlapping phases
- **Milestone Markers**: Key dates highlighted

### Prompt Template
```
Roadmap slide showing [NUMBER] phases.
Phases: [PHASE 1: TIMING], [PHASE 2: TIMING]...
Current position: [PHASE].
Style: [HORIZONTAL/VERTICAL] timeline.
Background: [COLOR], current phase highlighted in [ACCENT].
```

---

## 11. Pricing / Investment

**Purpose**: Present cost and value alignment

### Required Elements
- Pricing tiers or total investment
- What's included
- Optional: ROI calculation

### Visual Approaches
- **Pricing Cards**: Side-by-side tiers
- **Single Investment**: Focused total with breakdown
- **ROI Calculator**: Input/output visualization
- **Value Stack**: Everything included

### Content Principles
- Anchor high, justify value
- Include everything (no hidden costs)
- Connect to outcomes, not features

---

## 12. Next Steps / CTA

**Purpose**: Drive action and close

### Required Elements
- Clear call-to-action
- Specific next steps (numbered)
- Timeline for action
- Contact information

### Visual Approaches
- **Numbered Steps**: 1-2-3 action list
- **Single CTA**: One bold action
- **Timeline**: Immediate next steps
- **Contact Card**: How to reach out

### Content Principles
- Make it easy (remove friction)
- Create urgency (timeline)
- Be specific (what exactly to do)

### Prompt Template
```
Call-to-action slide with [NUMBER] next steps.
Steps: [STEP 1], [STEP 2], [STEP 3]
CTA: "[ACTION PHRASE]"
Contact: [EMAIL/PHONE/LINK]
Background: [COLOR], CTA button in [ACCENT].
Minimal, focused, action-oriented.
```

---

## 13. Closing / Thank You

**Purpose**: Memorable ending, contact info

### Required Elements
- Thank you or closing statement
- Contact information
- Logo/branding

### Visual Approaches
- **Minimal**: Clean with contact info
- **Quote**: Memorable closing thought
- **Call-back**: Reference to opening
- **Visual**: Impactful image with overlay

### Prompt Template
```
Closing slide for [COMPANY/PROJECT].
Message: "[CLOSING STATEMENT]"
Contact: [EMAIL], [PHONE], [WEBSITE]
Logo: [POSITION]
Background: [COLOR], elegant minimal style.
```

## Slide Sequencing

### Standard Narrative Flow
1. Title → Hook attention
2. Problem → Create tension
3. Solution → Relieve tension
4. How It Works → Build understanding
5. Benefits → Build desire
6. Proof → Build credibility
7. Investment → Address cost
8. Next Steps → Drive action
9. Closing → Memorable end

### Pitch Deck Flow (Investors)
1. Title
2. Problem
3. Solution
4. Market Size
5. Product Demo
6. Business Model
7. Traction
8. Team
9. Financials
10. Ask/Use of Funds

### Sales Deck Flow
1. Title (Client-focused)
2. Understanding (Their situation)
3. Challenges (Their pain)
4. Solution (Your offering)
5. How It Works
6. Results (Social proof)
7. Why Us (Differentiators)
8. Investment
9. Implementation Plan
10. Next Steps
